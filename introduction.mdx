SIC-HVD is an internal web application built for HVD staff to manage capital investment requests (SIC) and memo requests from creation through final approval. Originators draft and submit requests, approvers review and act on them, and super admins oversee the entire system — all within a single Filament-powered admin panel that handles notifications, PDF generation, and audit history automatically.

## System overview

SIC-HVD uses three roles managed by [Filament Shield](/reference/filament-panel) (powered by Spatie Laravel Permission). Each role has a distinct set of permissions that determines what a user can see and do in the panel.

| Role | Responsibilities |
|---|---|
| **originator** | Creates and submits capital investment and memo requests. Can edit drafts and resubmit returned requests. |
| **approver** | Reviews pending and resubmitted requests. Can approve, return, or reject requests and leave review comments. |
| **super_admin** | Full system access. Manages users, roles, companies, departments, and all request data. |

For a complete breakdown of what each role can access, see [Roles and permissions](/user-guide/roles-and-permissions).

## Request types

SIC-HVD handles two distinct request types, each with its own form, attachment rules, and approval path.

**Capital investment requests (SIC)** are used to request approval for spending on capital assets or significant operational investments. They require cost justification, departmental attribution, and supporting documentation. Once approved, the system generates a signed PDF record.

**Memo requests** are used for internal approvals that require documented justification but do not necessarily involve capital expenditure. They follow the same approval workflow as SIC requests and also produce a signed PDF on approval.

Both request types share the same status lifecycle: `draft` → `pending` → `approved` (or `returned` / `rejected`). A returned request can be revised and resubmitted, which moves it to the `resubmitted` state before re-entering the approval queue. See [Request statuses](/reference/request-statuses) for the full state machine.
